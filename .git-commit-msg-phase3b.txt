redesign: implement Dark Noir visual system with Stitch design tokens and Framer Motion

- Pull design tokens, structure, and component reference from Stitch 
  MCP (Dark Noir / Scout direction) rather than approximating visually
- Set dark theme as default (was prefers-color-scheme fallback only),
  light theme retained as secondary
- Fix WCAG AA contrast failure: on-surface-variant token (#4a4455 on
  #0F0F0F, ~2:1 ratio) corrected to #a8a2b1 (~7.7:1), preserving the
  intended violet-gray hue
- New StatTag component: ledger-style stat chips (Inter label +
  JetBrains Mono value) used across cards and detail page
- Layout: sticky header, Playfair Display wordmark, restyled shortlist
  trigger
- PlatformTabs (renamed from PlatformFilter): pill-style tabs; search
  input kept as its own separate component, not merged in
- ProfileGrid: CSS grid layout (12/6/4 col span across breakpoints)
  replacing the old flex layout
- ProfileCard: full-bleed image with gradient overlay, Playfair
  heading, StatTag grid; focus-visible ring now uses an offset so it
  stays visible against image backgrounds
- ProfileDetailPage: restructured around the ledger/stat-grid aesthetic
- ShortlistDrawer: solid surface styling matching the Stitch spec
- Added framer-motion: spring slide-in on the drawer, hover glow +
  scale on cards, micro-pulse on Add to List toggle, shimmer skeleton
  loaders for images
- All animations respect prefers-reduced-motion by disabling
  spring/scale/slide outright (not just shortening duration)
- Icons via lucide-react rather than a full icon font, for bundle size
- npm run build and npm run lint pass clean
